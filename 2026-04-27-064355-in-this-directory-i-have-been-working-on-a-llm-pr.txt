❯ This may not work as expected, because the model was trained with absolute    
  position embeddings. So, I would only expect it to respond correctly if it    
  sees "Question:" as the first token. Any thoughts?                            

● Good observation, but the current chat.py actually handles this correctly —
  each call to generate() starts fresh with "Question: ...\nAnswer: " at
  position 0, exactly as during SFT training. There's no accumulated history.

  The concern would only matter if you wanted multi-turn context — e.g. feeding
  previous Q&A pairs into the prompt so the model can refer back to them. That
  would shift "Question:" away from position 0 and likely degrade quality. For
  now, each turn is independent, which is the safe choice given the absolute
  position embeddings.

  If you ever want to experiment with multi-turn context despite that
  limitation, one mitigation would be to fine-tune with multi-turn examples so
  the model sees "Question:" at various positions. But for this wrapper, the
  stateless approach is the right default.
